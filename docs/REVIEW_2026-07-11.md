# Codebase Review — 2026-07-11

Full-depth review of `src/`, `tests/`, `scripts/`, and the five notebooks, driven by
four dimensions: (1) fragility/speed of feature & label preparation, (2) numerical
safety of numeric/ML code, (3) strength of the domain model, (4) architectural
consistency. Every claim below carries a `file:line` reference or a measured number.

This document records **what is**, ranked by severity. The target design lives in
[TARGET_ARCHITECTURE.md](TARGET_ARCHITECTURE.md).

---

## 0. Measured evidence

| Measurement | Result |
|---|---|
| `construct_labels_pl` (1-min cadence, M=20) | **3.40 s @ 527k rows** — Python `for k in range(K)` loop |
| Vectorized label kernel (same op order) | **0.047 s @ 527k rows, bit-exact** (72× speedup; verified `y`/`m_k`/`tau_k` `array_equal` incl. NaN) |
| `run_pipeline` 1-min, no derivatives | 29.3 s @ 60k rows → 39,821×1,603 frame (~0.49 ms/row) |
| Live rolling serving (from docs/ENGINE.md §8, real data) | **~20.4 s of feature compute per bar** on the 40,320-row buffer — the whole pipeline recomputes to produce one row |
| Batch serving | ~3.3 ms/bar |

The label loop and the per-bar full-pipeline recompute are the two structural
speed defects. The batch feature engine itself (polars expressions per tier) is
adequately fast; it is the *orchestration around it* that is slow.

## 1. Feature & label preparation — fragile and slow

**L1. Label computation is a Python row loop.** `src/features/boundary.py:132-165`
iterates `for k in range(K)` with per-iteration numpy slicing. At 1-min cadence K is
every bar (527k/year). Measured 3.4 s vs 0.047 s vectorized (bit-exact —
`sliding_window_view` + same divide-then-log operation order). The loop also
allocates 2 numpy slices per row.

**L2. Positional-alignment contract between two frames.** `construct_labels_pl`
takes `df_boundaries` + `df_raw` and correlates them by `n_k = k * bar_stride`
index arithmetic (`boundary.py:133`). Nothing validates that the frames are
actually row-aligned (no timestamp cross-check); a caller that slices one frame
but not the other gets silently wrong labels. `run_inference_pipeline` works
around this by manually tail-slicing both frames together (`pipeline.py:294-303`)
— the invariant lives in a comment, not in code.

**L3. Three dataframe dialects in one path.** The pipeline is pandas in →
`compute_base_series` (pandas, legacy `utils.py:1521`) → polars engine →
boundary stages that internally round-trip **numpy and pandas** (`boundary.py:610-615`
uses `pd.Series(...).rolling(...).std(ddof=0)` inside a polars function "for
legacy parity"; same at 673-724). Every conversion is a copy plus a
semantics boundary (NaN vs null, tz handling) that has already produced real
bugs (`_coerce_expected_numeric` in `pipeline.py:114-157` exists to patch
`pl.from_pandas` dtype damage after the fact).

**L4. Feature-set membership is defined by string subtraction against private
constants.** The model's feature list = "all columns except `_LABEL_AUX_COLS ∪
_RAW_COLS ∪ _BASE_COLS ∪ _DERIV_BASE_COLS`" (`pipeline.py:497-498`). Notebook 02
and `src/engine/retrain.py:47-53` both **import those underscore-private tuples**
and repeat the set algebra. A new non-feature column added to the frame silently
becomes a model feature everywhere unless four call sites are updated.

**L5. Imputation policy is a 140-line order-sensitive regex table keyed on column
names** (`utils.get_imputation_value`, `utils.py:2804-2951`), ending in a silent
catch-all `(r".*", 0.0)`. The Feature classes that emit the columns declare
`expected_range`/`max_nan_rate_after_warmup` (`features/base.py:62-65`) but NOT
their imputation value — the two halves of one contract live 2,000 lines apart in
different files, joined by regex. A new family gets `0.0` imputation silently.

**L6. Configuration is module-level globals frozen at import time.**
`features/config.py` re-exports 44 window constants from `utils.py`; family
classes bind them at class-definition time (`equilibrium.py:74`
`windows = tuple(WINDOWS_EQ)`; `equilibrium.py:55` `_SQRT_M = math.sqrt(float(M))`).
Consequences: two configurations cannot coexist in one process, tests must
monkeypatch modules, and changing M requires editing source. `_PipelinePlan`
(`pipeline.py:314-386`) is a good start at run-scoped config but still reads the
globals underneath.

**L7. Warmup/trim bookkeeping is manual and duplicated.** `K_WARMUP` vs `N_WARMUP`
row-unit conversions appear in `pipeline.py:236-244`, `features/engine.py:106-115`,
engine `FeatureContract.n_warmup`, and notebook 02's health gates — four places
that must agree.

## 2. Numerical safety — concrete defects

Crash-on-edge-case (reachable today):

**N1.** `src/analytics/edge.py:224` — `(point[:, 2] * len(y)).round().astype(int)`
crashes `ValueError: cannot convert float NaN to integer` when a split is
single-class (the sweep matrix is all-NaN by design at `edge.py:115-116`).

**N2.** `src/analytics/metrics.py:90` and `degradation.py:547` — `pd.qcut(regime, 3)`
without `duplicates="drop"`: raises on tied/point-mass regime values, which
vol-regime columns produce routinely.

**N3.** `src/analytics/degradation.py:92-108` — `psi()` divides by
`len(p_reference)` / `len(p_current)` with no empty guard.

Silent-corruption (worse than crashes):

**N4. NaN `phi` opens a position that can never take profit.**
`simulator.py:544` computes `tp_price = bar_close * math.exp(phi)` from a
per-row cache value that is never finiteness-checked (`simulator.py:333`);
`Position.__post_init__`'s guard `if self.tp_price <= 0` (`inventory.py:44`) is
**False for NaN**, so the NaN position is accepted, `high >= NaN` never fires,
and the trade silently sits until expiry. The live path validates `contract.phi`
(`engine/features.py:72`) but the offline simulator does not.

**N5.** `src/analytics/edge.py:142-145` — Sharpe variance via naive
`E[X²]−E[X]²` on cumulative sums: catastrophic cancellation for small-dispersion
returns, `ddof=0`, negatives silently masked to NaN by the `var_r > 1e-18` guard.

**N6.** `src/analytics/uncertainty.py:123` — `1/(1+exp(-x))` overflows for large
negative logits (RuntimeWarning + potential inf); `scipy.special.expit` is the
stable form.

**N7. Inconsistent NaN contract across bootstrap aggregators.**
`degradation.py:257-259` and `cohorts.py:337-338` use `np.quantile` (one NaN
replicate poisons the CI) while every other module uses `np.nanquantile` +
`B_effective`. Same concept, two silent behaviors.

**N8. Over-broad `except ValueError: pass`** in bootstrap loops
(`bootstrap.py:196`, `curves.py:227,287,348`, `edge.py:297,339`) — intended for
single-class resamples but swallows any ValueError from the metric fn, silently
dropping replicates with no log line.

**N9.** Float truthiness on prices: `engine.py:783-785`
(`last_close or float("nan")`) treats a 0.0 close as missing; float-equality
repair detection at `guards.py:158-160`.

**N10. Closure-state leak in exit policy.** `make_exit_let_winners_run_monotonic`
(`policy.py:507-563`) keeps `p_max_per_position` in a closure, popped only on
`"expiry"`/`"tp_market"`. Bulk-close and SL paths bypass `exit_policy` entirely
(`simulator.py:436` `portfolio.close_all`), so entries leak forever; state also
silently resets on live hot-swap — an offline/live divergence.

**N11. Synthetic gap-repair bars silently enter retraining.** The `synthetic`
flag exists on `Bar` (`engine/domain.py:58`) and is persisted, but
`store.bars_frame()` drops the column (`store.py:469`) before
`run_retrain_job` consumes the frame — flat synthetic bars train the model
indistinguishably from real bars.

Guarded well (worth keeping as exemplars): `audits.deflated_sharpe`
(`audits.py:452-473`), weight functions' log-space caps (`utils.py:3052-3077`,
3159-3169), `_kelly` (`policy.py:174-177`), `matured_label`
(`engine/features.py:323-329`), post-impute triple assertion (`quality.py:124-142`).

## 3. Domain model — weak where it matters

**D1. The strategy cache is a stringly-typed DataFrame.** `simulate()` reads
`row["k"], row["p"], row["regime"], row["phi"], row["close"]` per row
(`simulator.py:329-348`) from an implicit schema documented only in docstrings
and enforced nowhere. `fast_train.CACHE_REQUIRED_COLS` (`fast_train.py:35`)
defines the schema but no consumer imports it; `edge`/`degradation`/`cohorts`
hardcode the same column literals independently.

**D2. Two half domain models that never met.** `src/engine/domain.py` has
`Bar/Order/Fill/Trade/Decision` (frozen, slots — good); `src/strategy/inventory.py`
has `Position/ClosedPosition/Portfolio` (invariant-checked — good). But the
offline simulator never uses `Fill`/`Order`; exit pricing is re-derived inline
(`simulator.py:188-194` vs `execution.py`), and `Trade` duplicates
`ClosedPosition` semantics.

**D3. Missing first-class concepts:** `Clock`/grid (tz-stripping and M-grid
arithmetic repeated at `cache.py:55,64`, `simulator.py:134-144`, `baseline.py:30-38`,
`reporting.py:277-287`, `buffer.py:33`); `Instrument`/cost model (`phi`, `M`,
`cost_per_trade` travel as loose floats across `FeatureContract`, `SimConfig`,
`RiskConfig`, and per cache-row); `Cluster` (5-7 loose local variables duplicated
in `simulator.py:311-318` and `engine/strategy.py:204-211`); `Ledger` (smeared
across `Portfolio.closed_positions`, `Trade`, SQLite); `Signal` (a bare float
`p`); `ExitReason` (unchecked strings wired into ≥5 sites — `inventory.py:22`,
`simulator.py:188-194`, `reporting.py:105-109,367-373`, `__main__.py:117`).

**D4. StrategySpec bundles closures — uninspectable, unserializable.** Specs are
built as `lambda s, t=P_TH: gate_score_above(s, t)` (`policy.py:670-965`,
notebook 05). A spec cannot be printed, diffed, hashed, or persisted; its
parameters are trapped in closure cells. The notebook sweep re-builds the whole
25-line lambda block per grid point.

**D5. `State` is an 18-field god-parameter** (`policy.py:42-78`) reconstructed
3-4× per boundary, including via `State(**{**vars(state_no_score), "score": s})`
(`simulator.py:528`). Every primitive receives all 18 fields regardless of need.

**D6. The boundary step exists twice.** `simulate()` (`simulator.py:327-622`)
and `LiveTrader.on_boundary` (`engine/strategy.py:215-396`) hand-mirror the same
10-step loop (exits → bulk → expiry → decide, State composition, cluster
bookkeeping, TP-price formula at `simulator.py:544` = `strategy.py:355`).
Parity is maintained **by test, not by construction** — every new strategy
feature is a two-file, parity-fragile edit.

## 4. Architectural consistency

**A1. `src/utils.py` is a 3,425-line god module** — config constants, JSON I/O,
Binance URL generation, ZIP loaders, checkpoint printers, ~40 legacy pandas
`compute_*` feature functions (now superseded by `src/features/` but still the
imputation/oracle dependency), label construction, training weights, splits,
metrics. Notebooks consume exactly 21 functions + 16 constants of it; the rest
is dead weight or parity oracle.

**A2. Config drift is live in the repo.** Notebook 03's markdown documents
`l2_leaf_reg=5.0, random_strength=1.5` while its code has `l2_leaf_reg: 0.1,
random_strength: 0.0`. Hyperparameters, regime column name
(`REGIME_COL='vol__rs__f__w240'`), thresholds, and lot sizes live as notebook
literals re-typed across 03/04/05 and two scripts.

**A3. Cross-package private-symbol imports.** `engine/retrain.py:47-53` imports
four `_`-private tuples from `features/pipeline`; notebook 02 does the same;
`engine.py:288` reaches into `store._conn`; `edge.py:42` imports private
`_interp_pr/_interp_roc` from `curves`; `audits.py:532` imports `fast_train`
inside a function to dodge a circular dependency.

**A4. Duplication.** Bootstrap resample scaffolding copy-pasted ~8× across
analytics; `_choose_indices` exists 4× verbatim (`curves.py:47`,
`degradation.py:52`, `edge.py:46`, inline in `bootstrap.py:187`); notebook 01
duplicates its own download loop inline; scripts duplicate cache-loading and
spec-construction blocks verbatim (`run_strategy_variants.py:80-103` =
`run_winning_strategy_charts.py:98-121`); wilson_interval lives in
`degradation` but is imported by `edge` and `uncertainty`.

**A5. Compute and presentation interleaved.** Six analytics modules each end in
a matplotlib/seaborn `plot_*` tail; notebook 04 then **re-implements** those
plots by hand (~120 lines/figure) instead of calling them.

**A6. Silent observability gap.** Zero `logging` in `src/analytics/`,
`src/features/`, `src/strategy/` (the engine logs well). Long bootstrap/
permutation routines emit nothing; pipeline stages emit nothing (stage timing,
row counts, trim decisions are all invisible).

**A7. Notebook ergonomics tax ≈ 2,300 lines.** Every notebook re-resolves the
repo root, re-declares artifact paths as string literals, revalidates its
inputs by hand (03's 90-line integrity gate), and hand-assembles provenance
dicts. The artifact contract between notebooks is filenames-by-convention.
`SimResult` needs ~250 notebook lines of post-processing (headline table,
composition reconstruction, dashboards) that belong on the result object.

## 5. What is strong (keep, and build on)

- **Parity machinery**: `tests/engine/test_parity_simulator.py` (live≡offline
  ledgers, exact), `test_feature_inference.py` (serve≡train features),
  `test_engine_e2e.py` (resume≡uninterrupted, kill-switch, degrade→halt→recover).
  This is the safety net that makes re-architecture feasible.
- **Feature-as-class registry** (`features/base.py`) — declarative polars
  expressions, tiered execution, collision detection at construction
  (`features/engine.py:54-69`). Right bones, wrong config binding.
- **Primitives with documented pandas-parity semantics** (`features/primitives.py`)
  and the polars-landmine discipline around them.
- **`analytics/sampling.py`** — purged splits + uniqueness weights;
  linear-time, validated, algorithmically excellent. Same for
  `thresholds.py` and `fast_train`'s atomic cache I/O.
- **Typed error taxonomy + guards in the engine** (`engine/errors.py`,
  `guards.py` — every repair emits a `GuardEvent`), atomic model registry
  (`model.py:207-247`), store flush that retains rows on failure
  (`store.py:294-312`).
- **Causal discipline as a cultural norm**: exit→bulk→expiry→decide ordering
  documented and tested; "rank against past, then absorb" online stats
  (`online.py:74-79`); label-maturity shifts (`boundary.py:187-215`).
- **RNG hygiene** in analytics (universal `default_rng(seed)`, no global state).

## 6. Ranked master list

1. Boundary step duplicated between offline simulator and live trader (D6) —
   every strategy change is a parity-fragile two-file edit.
2. Label computation: Python loop + unvalidated positional alignment (L1, L2).
3. Live serving recomputes the full pipeline per bar (~20 s/bar) — structural,
   not incidental (§0).
4. Config as import-time module globals; two configs cannot coexist (L6, A2).
5. Stringly-typed frame contracts everywhere: cache columns, feature-set
   membership by private-constant subtraction, exit reasons (D1, L4, D3).
6. Silent numerical corruption paths: NaN phi position (N4), synthetic bars in
   retraining (N11), closure-state leak (N10), NaN-contract drift (N7).
7. Imputation-by-regex divorced from Feature declarations (L5).
8. `utils.py` god module + cross-package private imports (A1, A3).
9. Crash edges in analytics (N1, N2, N3) + duplicated bootstrap scaffolding (A4).
10. Spec-as-closures: strategies not inspectable/serializable/sweepable (D4).
11. Notebook ergonomics: 2,300 lines of boilerplate, config drift, no run/artifact
    objects (A7, A2).
12. No logging in research packages (A6); no lint/type/coverage gate in CI.
